# Example: Architecture Validation

## Scenario

You're designing a new system and want a second opinion on the architecture.

## Prompt

```
/ask-oracle Evaluate the database schema design in docs/schema.sql for scalability issues
```

## What Happens

1. Oracle reads the schema/design documents
2. Asks Codex to critique the architecture
3. Verifies Codex's feedback is relevant and actionable
4. Reports findings with specific improvement suggestions
